# [RSS Reader] 开发记录

## [v1.0.0] - 2026-06-19

**负责人/开发者**: Claude Code

**状态**: 开发中

### 🚀 新增特性 (Features)

* **项目初始化** 创建了基于 Electron + React + TypeScript + Vite 的桌面 RSS 阅读器项目骨架

* **RSS订阅管理** 实现了订阅源的添加、删除、刷新功能，支持 rss-parser 解析 RSS/Atom 格式

* **分类管理** 实现了分类的创建、重命名、删除，支持为订阅源分配分类

* **文章列表与阅读** 实现了三栏布局：侧栏（订阅源列表）→ 文章列表 → 阅读视图，支持文章列表筛选

* **已读/星标管理** 实现了文章的标记已读、标记全部已读、星标收藏功能

* **全文提取** 集成了 @extractus/article-extractor 和 Draft.js JSON 解析器，支持自动提取文章全文

* **OPML 导入/导出** 支持 OPML 格式的批量导入和导出备份

* **搜索功能** 支持按标题和摘要全文搜索文章

* **离线存储** 使用 better-sqlite3 + SQLite 实现本地离线存储

* **阅读体验优化** 点击文章自动收起侧栏，文章列表缩窄，内容区占满全屏

* **全局错误提示** 添加订阅源时显示友好的中文错误提示（重复添加、无效URL等）

### 🐛 问题修复 (Bug Fixes)

* **豆瓣 RSS 乱码** 修复了豆瓣影评 RSS 的 Draft.js JSON 格式解析问题，包括 HTML 实体解码、残缺 Unicode 转义补零、控制字符清理

* **全文提取可靠性** 将文章内容提取引擎从 @extractus/article-extractor(E模块) 替换为 Mozilla Readability(Firefox阅读模式引擎) + cheerio 双层兜底，大幅提升提取成功率

* **侧栏不刷新** 修复了添加订阅源后侧栏不自动刷新的问题，引入 feedListVersion 刷新信号机制

* **原生模块兼容** 修复了 better-sqlite3 在 Electron 环境中 NODE_MODULE_VERSION 不匹配的问题，通过 @electron/rebuild 重建

* **Electron 二进制缺失** 修复了 electron npm 包下载二进制失败的问题，手动提取并创建 path.txt

* **TypeScript 6 兼容** 修复了 moduleResolution 在 TypeScript 6.x 下的弃用警告

* **移除默认菜单栏** 移除了窗口顶部的 "File Edit ViewWindow" 默认菜单栏

* **全文提取架构重构** 将提取逻辑从前端异步调用( `autoExtract` )改为后端 `articles:get` 同步提取，彻底解决了部分文章提取不到的问题

### ⚡ 性能优化 (Optimizations)

* **数据库索引** 为 articles 表的 feed_id、is_read、is_starred、published_at 和 feeds 表的 category_id 添加了索引

* **增量更新** 刷新订阅源时使用 INSERT OR IGNORE 避免重复插入已有文章

### ⚠️ 破坏性变更 (Breaking Changes)

* 无（初始版本）

## [v1.1.x] - 2026-06-19

**负责人/开发者**: Claude Code

**状态**: 开发中

### 🚀 新增特性 (Features)

* **两栏布局重构** 从三栏（侧栏|文章列表|阅读视图）改为两栏（侧栏|内容区），内容区在文章列表和阅读视图间切换，不分割屏幕

* **内容选中复制** 文章区（标题、来源、正文）支持文字选中和 Ctrl+C 复制

### 🐛 问题修复 (Bug Fixes)

* **全文提取架构重构** 将提取逻辑从前端异步调用改为后端 `articles:get` 同步提取，彻底解决部分文章提取不到的问题

* **底部干扰内容过滤** 增加了 `cleanContent` 函数，自动移除提取结果中的"相关推荐""热门阅读"等干扰区块，保留原文链接

* **原文链接重复** 修复了原文链接后面的推荐内容显示问题，保留原文链接，移除其后兄弟元素

### ⚡ 性能优化 (Optimizations)

* **提取引擎替换** 从 `@extractus/article-extractor`(ESM) 替换为 `Mozilla Readability` + `linkedom` + `cheerio` 双层方案，兼容 CJS 环境

### ⚠️ 破坏性变更 (Breaking Changes)

* 无
